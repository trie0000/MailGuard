# MailGuard セットアップ ガイド (利用者向け)

メール誤送信検出ツールの初回セットアップ手順です。

## 必要なもの

- Windows 10 / 11 (= PowerShell 5.1 標準同梱)
- AI API キー (= 管理者から取得)

Node.js / npm は **不要** です。

## ① 配布フォルダを展開

管理者から受け取った ZIP / 共有フォルダを任意の場所に解凍 (例: `C:\Users\<自分>\MailGuard\`)。
中身は以下のような構成:

```
MailGuard/
├── dist/
│   └── mailguard.html              ← ブラウザで開く UI 本体
├── relay/
│   └── mailguard-relay.ps1         ← PowerShell 製の loopback プロキシ
├── MailGuard.bat                 ← 起動用バッチ
├── .env                            ← 組織共通設定 (admin 設定済み)
└── SETUP.md                        ← このファイル
```

## ② 起動

`MailGuard.bat` を **ダブルクリック**。

正常起動すると:
1. 黒い cmd ウィンドウが開いて relay の起動ログが表示される
2. 既定ブラウザで `mailguard.html` が自動で開く

cmd ウィンドウは **閉じないでください** (= 閉じると relay が止まります)。

## ③ API キーを設定

ブラウザの MailGuard 画面で:
1. 右上「⚙ 設定」をクリック
2. **API キー** 欄に管理者から受け取ったキーを入力
   - Anthropic Claude の場合: `sk-ant-...`
   - 社内 AI ゲートウェイの場合: 管理者発行のキー
3. 「保存」をクリック

その他の項目 (= プロバイダ / 上流 URL / モデル / 自社ドメイン 等) は
**管理者が `.env` で設定済み** のため、変更不要です。

## ④ 使う

1. Outlook で **下書きメールを作成**
2. メール一覧 / メッセージ ウィンドウから MailGuard 画面に **ドラッグ&ドロップ**
   - うまく行かない時は **メッセージのソースを表示** (= ⌥⌘U / Alt+U) → 全選択コピー →
     MailGuard 画面で **⌘V / Ctrl+V** で貼り付け
3. プレビュー表示を確認 → **「🤖 AI で誤送信チェック」** ボタン
4. リスク判定結果が表示される (= 🔴 高 / 🟡 中 / 🟢 低 / ✅ ok)
5. 高リスクの場合は内容を確認してから送信

## トラブル時

### relay が起動しない

`MailGuard.bat` の黒画面にエラーが出ます。よくあるケース:

- **二重起動**: 別の cmd で既に起動中。「ポート 18100 は既に別プロセスが使用中」と出るので、
  指示通り Ctrl+C で既存を止めるか、強制終了
- **PowerShell が無い**: 通常 Windows に標準で入っていますが、無効化されていれば管理者へ
- **PowerShell スクリプトのブロック**: `.bat` 経由なら自動回避されるはず

### AI チェックで「タイムアウト」

社内プロキシ環境では追加設定が必要。`.env` に以下を追記:
```
MAILGUARD_AI_PROXY=http://proxy.example.com:8080
```
プロキシ URL は IT 部門に確認。

### 設定が壊れた / 戻したい

ブラウザの **設定モーダル** → **「env デフォルトに戻す」** ボタンで、admin が `.env` に
設定した値に再設定できます。API キーは保持されます。

完全リセットしたい場合は、ブラウザの DevTools (F12) → Console で:
```js
localStorage.removeItem('mailguard.settings.v2')
location.reload()
```

## 更新方法

管理者から **新しい配布フォルダ** を受け取ったら、古いフォルダ ごと差し替えれば OK。
ブラウザ設定 (= API キー) は localStorage に残っているので失われません。
